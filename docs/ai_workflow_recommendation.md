# Recomendação de uso de modelos de IA para o projeto OpenClaw

## Objetivo

Organizar qual tipo de modelo de IA usar em cada etapa do projeto, especialmente quando modelos mais avançados estiverem disponíveis para revisão e evolução do que já foi construído.

## Estratégia recomendada

### 1. Modelos Generativos
Usar para:
- gerar código inicial
- gerar testes
- escrever documentação
- criar prompts e templates

Exemplos:
- ChatGPT
- Claude
- Gemini

### 2. Modelos de Busca e Síntese
Usar para:
- pesquisar soluções e padrões
- resumir documentação técnica
- comparar abordagens antes de implementar

Exemplos:
- Perplexity
- Consensus

### 3. Sistemas Agênticos
Usar para:
- executar múltiplos passos automaticamente
- rodar testes e validações
- iterar em ciclos de desenvolvimento
- trabalhar com terminal, arquivos e execução de workflows

Exemplos:
- Cursor
- Devika
- CrewAI

## Sequência recomendada de uso

1. Pesquisa e entendimento
   - usar modelos de busca e síntese
   - compreender o problema e as melhores práticas

2. Geração inicial
   - usar modelos generativos
   - criar estrutura, código inicial e documentação

3. Validação e execução
   - usar sistemas agênticos
   - rodar validações, testes e ajustes automáticos

4. Revisão e refinamento
   - usar modelos mais avançados para revisar o que já foi produzido
   - avaliar qualidade, consistência e evolução do projeto

## Recomendação para revisão do que já foi feito

Quando houver modelos mais avançados, a melhor sequência é:

1. pedir uma revisão arquitetural do projeto
2. pedir uma análise de gaps e riscos
3. pedir uma proposta de evolução do fluxo low-code
4. pedir uma revisão de documentação e estrutura
5. pedir uma sugestão de próximos passos com prioridade

## Modelo recomendado para esta revisão

Para revisar o que já foi construído, o melhor uso é:
- um modelo avançado de tipo generativo para revisão e sugestão de melhorias
- em seguida um agente para validar essas sugestões no ambiente real

## Padrão de uso para o projeto OpenClaw

- usar IA generativa para criar
- usar IA de busca para fundamentar
- usar agentes para executar e validar
- usar modelos avançados para revisar e elevar a qualidade do projeto
